Passing varialbes and parameters in numerical computation/simulation

Posted on April 17, 2020 in Research

When we write a computation program with a quite few subfunctions to call, we will need to choose how to elegently pass the varialbes and parameters.First, we want to distinguish the varialbs and paramters.

There is a clear difference between variables and parameters. A variable represents a model state, and may change during simulation. A parameter is commonly used to describe objects statically. A parameter is normally a constant in a single simulation, and is changed only when you need to adjust your model behavior.

In general, varialbes should be passed as function arguments. We now focus on how to elegently pass only required parameters. There are a few ways: 1. Global varialbes 2. Function arguments 3. Nested function 4. Object oriented instance variables

Nested function is what I recommended for the function-based programming for simple simulation (number of funcitons is less than 10 as a rule of thump), and we should use object oriented programming and thus instance varialbs to pass parameters for complicated simulation.

Nested Function in Matlab

Use Matlab as an example. Write the main code as a function and put the subfunctions inside of the main function will allow us to define parameters can be used as in those subfunctions. Then the arguments of these function can just pass the varialbs. Based on the harachy of the nesting --- two subfunctions can be put in parallel or one subfunction inside the other to allow one subfunction get access to the other function's internal varialbs or not. I feel this is a 'weak' version of instanc variables in objected programming.
